## Пункт 4. `_check_lockout` — неточный подсчёт неудачных попыток

**Файл:** `services/auth/auth.py`

Проблема в том, что `LIKE '%username=admin%'` может совпасть с `username=admin_backup`. Есть два варианта исправления.

**Вариант A (быстрый) — уточнить LIKE-паттерн:**

**Было:**
```python
def _check_lockout(username):
    with get_db() as conn:
        cutoff = time.time() - LOCKOUT_DURATION
        row = conn.execute(
            """SELECT COUNT(*) as cnt FROM audit_log
               WHERE action = 'login_failed'
               AND details LIKE ?
               AND created_at > datetime(?, 'unixepoch')""",
            (f'%username={username}%', cutoff)
        ).fetchone()
        if row and row['cnt'] >= MAX_LOGIN_ATTEMPTS:
            return True, LOCKOUT_DURATION
    return False, 0
```

**Стало:**
```python
def _check_lockout(username):
    with get_db() as conn:
        cutoff = time.time() - LOCKOUT_DURATION
        row = conn.execute(
            """SELECT COUNT(*) as cnt FROM audit_log
               WHERE action = 'login_failed'
               AND (details = ? OR details LIKE ? OR details LIKE ? OR details LIKE ?)
               AND created_at > datetime(?, 'unixepoch')""",
            (
                f'username={username}',          # точное совпадение
                f'username={username},%',         # в начале списка
                f'%, username={username},%',      # в середине
                f'%, username={username}',        # в конце
                cutoff
            )
        ).fetchone()
        if row and row['cnt'] >= MAX_LOGIN_ATTEMPTS:
            return True, LOCKOUT_DURATION
    return False, 0
```

**Вариант B (правильный) — записывать username в отдельное поле и искать точно:**

Шаг 1. Измените функцию `log_audit`, добавив параметр `username`:

## Пункт 9. Загрязнение `sys.path` множественными `insert(0, ...)`

**Проблема:** каждый сервис вставляет пути в начало `sys.path`, что может привести к импорту «чужих» модулей. Например, `websearch.py` вставляет путь к `generator/`, и если в директории `generator/` есть файл с именем, совпадающим с каким-то стандартным модулем, он перехватит импорт.

**Решение:** заменить `sys.path.insert(0, ...)` на `sys.path.append(...)` и использовать только тогда, когда действительно необходимо.

**Файл `services/websearch/websearch.py`:**

**Было:**
```python
generator_path = os.path.join(os.path.dirname(os.path.dirname(__file__)), 'generator')
sys.path.insert(0, generator_path)
# ...
sys.path.insert(0, str(Path(__file__).parent.parent / 'common'))
```

**Стало:**
```python
# Добавляем пути для импорта proto-файлов и общих модулей
# Используем append вместо insert(0), чтобы не перекрывать стандартные модули
_common_path = str(Path(__file__).parent.parent / 'common')
_generator_path = str(Path(__file__).parent.parent / 'generator')

if _common_path not in sys.path:
    sys.path.append(_common_path)
if _generator_path not in sys.path:
    sys.path.append(_generator_path)
```

**Файл `services/web_ui/web_ui_service.py`:**

**Было:**
```python
sys.path.insert(0, str(Path(__file__).parent.parent / 'common'))
```

**Стало:**
```python
_common_path = str(Path(__file__).parent.parent / 'common')
if _common_path not in sys.path:
    sys.path.append(_common_path)
```

**Файл `services/webconfig/webconfig.py`:**

**Было:**
```python
sys.path.insert(0, str(Path(__file__).parent.parent / 'common'))
```

**Стало:**
```python
_common_path = str(Path(__file__).parent.parent / 'common')
if _common_path not in sys.path:
    sys.path.append(_common_path)
```

Тот же паттерн нужно применить во всех файлах, где встречается `sys.path.insert(0, ...)`. Ключевые правила: использовать `append` вместо `insert(0)`, проверять `if path not in sys.path` перед добавлением.

## Пункт 10. Неверный `current_pos` в chunker

**Файл:** `services/chunker/chunker.py`

Проблема: при наличии overlap чанки перекрываются, и `current_pos += len(chunk_text)` не соответствует реальному смещению в исходном тексте.

**Было:**
```python
current_pos = 0
for chunk_text, _ in splitter.split_text(text):
    meta = {"source": filename, "length": str(len(chunk_text))}

    yield chunker_pb2.Chunk(
        text=chunk_text,
        start=current_pos,
        end=current_pos + len(chunk_text),
        metadata=meta
    )
    current_pos += len(chunk_text)  # Примерная позиция
```

**Стало:**
```python
for chunk_text, _ in splitter.split_text(text):
    # Находим реальную позицию чанка в исходном тексте
    # Ищем начиная с позиции после начала предыдущего чанка,
    # чтобы корректно обрабатывать повторяющиеся фрагменты
    start_pos = text.find(chunk_text, getattr(self, '_last_search_pos', 0))
    if start_pos == -1:
        # Фолбэк: если точное совпадение не найдено (например, текст был
        # модифицирован сплиттером), ищем с начала
        start_pos = text.find(chunk_text)
    
    if start_pos == -1:
        # Крайний случай: chunk_text отсутствует в оригинале
        # (возможно при нормализации пробелов сплиттером)
        start_pos = 0
        end_pos = len(chunk_text)
    else:
        end_pos = start_pos + len(chunk_text)
    
    # Запоминаем позицию для следующего поиска
    # Сдвигаемся на (размер чанка - overlap), чтобы не пропустить следующий
    self._last_search_pos = start_pos + 1

    meta = {"source": filename, "length": str(len(chunk_text))}

    yield chunker_pb2.Chunk(
        text=chunk_text,
        start=start_pos,
        end=end_pos,
        metadata=meta
    )
```

Однако использование `self._last_search_pos` здесь неудобно, так как `ChunkIt` — это метод, вызываемый для каждого запроса. Лучше использовать локальную переменную:

```python
last_search_pos = 0
for chunk_text, _ in splitter.split_text(text):
    start_pos = text.find(chunk_text, last_search_pos)
    if start_pos == -1:
        start_pos = text.find(chunk_text)
    
    if start_pos >= 0:
        end_pos = start_pos + len(chunk_text)
        last_search_pos = start_pos + 1
    else:
        start_pos = 0
        end_pos = len(chunk_text)

    meta = {"source": filename, "length": str(len(chunk_text))}

    yield chunker_pb2.Chunk(
        text=chunk_text,
        start=start_pos,
        end=end_pos,
        metadata=meta
    )
```

Если `split_text` уже возвращает позицию во втором элементе кортежа (сейчас она игнорируется через `_`), стоит проверить, что именно возвращает `RecursiveChunker.split_text`. Если второй элемент — это позиция, просто используйте его:

```python
for chunk_text, position in splitter.split_text(text):
    start_pos = position if position is not None else text.find(chunk_text)
    end_pos = start_pos + len(chunk_text) if start_pos >= 0 else len(chunk_text)
    # ...
```
